# 当前 WG VPN 状况

## 适用场景

用户想查看当前 WireGuard VPN 是否在线、是否握手、peer 是否存在、服务端是否运行时，使用本模块。

## 固定入口

并行调用：

1. `openclaw_get_wg_status`
2. `clawwrt_list_devices`

## 执行流程

1. 先展示服务端状态结果。
2. 若在线设备数为 0：停止，并提示用户当前没有可查询的在线路由器。
3. 若用户未指定设备：
   1. 展示在线设备清单。
   2. 询问用户要查看哪些设备的 WG 状态。
   3. 未确认前不继续。
4. 对用户确认的每台设备，顺序调用 `clawwrt_get_wireguard_vpn_status`。
5. 输出时明确区分：
   1. 服务端状态
   2. 路由器侧接口/peer/握手状态
   3. 若有异常，指出异常位于哪一侧

## 输出要求

1. 不把“当前 Agent 运行环境的服务端检查结果”擅自表述成“VPS 一定正常”。
2. 若服务端结果不可用，只能说“当前服务端检查失败或不可用”，不能推断 VPN 一定故障。
3. 若路由器返回有握手时间、收发流量，应明确写出“隧道已有握手/有流量”。
