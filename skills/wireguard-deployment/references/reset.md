# 重置当前 WG VPN 配置

## 适用场景

用户要求清空、重置、回滚当前 WireGuard VPN 配置时，使用本模块。

## 固定入口

并行调用：

1. `openclaw_get_wg_status`
2. `clawwrt_list_devices`

## 执行流程

1. 先向用户说明重置分为两部分：
   1. VPS 服务端配置重置
   2. 路由器客户端配置重置
2. 必须让用户明确确认：
   1. 是否重置 VPS 服务端
   2. 要重置哪些设备的客户端配置
   3. 若用户说“重置全部/全部清空/恢复初始 WG 状态”，将其解释为“重置服务端 + 所有当前在线 WiFi 路由器”
3. 若用户要求重置 VPS 服务端：调用 `openclaw_reset_wg_server`。
4. 若用户要求重置客户端：
   1. 先展示在线设备清单。
   2. 对用户确认的设备逐台调用 `clawwrt_reset_wireguard_vpn`。
5. 若用户明确要求“重置所有”：
   1. 先调用 `openclaw_reset_wg_server`
   2. 再获取并展示当前在线设备清单
   3. 对所有当前在线 WiFi 路由器逐台调用 `clawwrt_reset_wireguard_vpn`
   4. 输出时明确标注“本次重置范围 = 服务端 + 全部在线路由器”
6. 任一步失败即停止该分支流程，并报告已完成部分。

## 规则

1. 不允许在未获确认的情况下同时重置所有在线设备。
2. 不因为服务端重置成功就默认客户端也应自动重置。
3. 不因为客户端重置成功就默认服务端 peer 已完全清空，除非对应 API 明确返回。
4. “重置所有”必须视为高风险操作，只有在用户明确表达“全部重置”时才能执行，不能由模型自行推断。

## 收尾

1. 重置完成后，建议用户再执行“验证当前 WG VPN 网络”模块。
2. 若用户明确要求立即验证，再进入 `references/verify.md`。
